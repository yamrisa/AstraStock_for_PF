## 📌 サイト名
**AstraStock（アストラストック）**  
“星のように散らばる在庫も一元管理する”というコンセプトで命名。

---

## 📌 サイトテーマ
**店舗向けの在庫管理・発注管理に特化した Web アプリケーション。**  
商品マスター、在庫ログ、発注ステータス、欲しいものリストなどをまとめて管理し、  
店舗全体の在庫把握と発注業務をスムーズにすることを目的としている。

---

## 📌 テーマを選んだ理由
- 店舗運営では紙ベースのカルテや手書きの発注メモが多く、管理が煩雑になりがち  
- 何をどれだけ使ったか曖昧になりやすい  
- 発注忘れ、重複発注、在庫過多などのトラブルが頻発する  
- ネイルサロン経験から「現場が本当に必要な在庫管理」の痛点が明確だった  

これらを解消するため、  
**発注ログを入力すれば注文票に自動計算される仕組み**  
**発注ステータスが可視化される仕組み**  
を一から構築することを選んだ。

---

## 📌 ターゲットユーザー
- 美容・ネイルサロンのオーナー・管理者  
- スタッフ人数が 2〜20 名ほどの小中規模店舗  
- 発注担当が固定されていない店舗  
- チームで使える、初心者でも迷わない在庫システムを求めている店舗  

---

## 📌 主な利用シーン
- 発注したい商材を「発注リスト」として即時記録、発注後は集計され期間別で発注数や金額の確認が可能
- 新しく使用したい商材やスタッフ個人の注文は「欲しいものリスト」「個人注文リスト」として記録を分ける 
- 一般スタッフは「発注リスト・欲しいものリスト・個人注文リスト」に追加のみ  
- 管理者は「未発注 → 発注済み → 入荷済み」へステータス管理と過去の発注金額・数量などの閲覧が可能 
- 店舗全体の在庫数・発注履歴を可視化

---

## 📌 機能一覧（Option）

### 管理者
- 商品マスター登録・編集  
- 在庫ログの閲覧（スタッフ別の消費数など）  
- 発注ステータス管理（未発注 / 発注済み / 入荷済み）  
- 店舗の発注金額集計・カテゴリ別分析  

### 一般スタッフ
- 商品検索（商品コード / カテゴリ / 写真一覧）  
- 発注リスト追加  
- 欲しいものリスト追加（新規入荷希望）  
- 重複発注チェック
